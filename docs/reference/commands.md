# Commands

Commands are slash-invoked skills. For tips on working with Lore, see [Working with Lore](../guides/working-with-lore.md).

## Command Reference

| Command | Description | Example Usage |
|---------|-------------|---------------|
| `/lore-capture` | Review session work, capture skills, update registries, validate consistency. | `/lore-capture` |
| `/lore-consolidate` | Deep health check: find stale items, semantic overlaps, knowledge drift. | `/lore-consolidate` |
| `/lore-status` | Show Lore version, hook health, skill counts, worker tiers, and active work. | `/lore-status` |
| `/lore-update` | Pull latest harness files from GitHub without touching operator content. | `/lore-update` |
| `/lore-link <target>` | Link a work repo so hooks fire from the hub. | `/lore-link ../my-app` |
| `/lore-docker` | Start, stop, or check the local Docker sidecar for semantic search and a live MkDocs UI. | `/lore-docker` |
| `/lore-create-note` | Create a lightweight note for quick capture. | `/lore-create-note` |
| `/lore-create-roadmap` | Create a strategic roadmap. | `/lore-create-roadmap` |
| `/lore-create-plan` | Create a tactical plan. | `/lore-create-plan` |
| `/lore-create-brainstorm` | Save a brainstorm for future reference. | `/lore-create-brainstorm` |
