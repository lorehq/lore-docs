# First Session Setup: Personal

!!! note "Coming soon"
    This guide is planned for a future release. In the meantime, [Knowledge Worker](knowledge-worker.md) covers many universal patterns — identity setup, service mapping, semantic search — that apply to personal deployments.

**Target profile:** Individuals using Lore for personal knowledge management, task tracking, research, and daily workflow — no cloud infrastructure, minimal CLI auth complexity, no enterprise org context.

**What will differ from Knowledge Worker:**

- Minimal CLI auth — typically a single GitHub account at most
- Secret store optional if no sensitive API keys are in scope
- Service mapping focused on personal tools: notes apps, cloud storage, communication, subscriptions
- Work context from personal goals and projects, not org OKRs
- Simpler model configuration — single tier may be sufficient for most tasks
- No org wiki discovery — public docs and personal notes are the knowledge sources
